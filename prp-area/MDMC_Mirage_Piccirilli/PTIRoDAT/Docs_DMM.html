Documentazione dello Script DMM.py

Overview

Questo script implementa un'interfaccia grafica (GUI) in PyQt6 per facilitare la creazione e documentazione di esperimenti su eLabFTW, a partire da file di misura in formato .ptir (HDF5). Include l'estrazione automatica di metadati, la generazione di grafici e immagini di anteprima, l'invio a eLabFTW, e la creazione di una copia FAIR dei dati.

Funzionalità principali

  • GUI User-Friendly per la selezione del template e file .ptir

  • Accesso via API REST a eLabFTW per creare e aggiornare esperimenti

  • Parsing dei campi extra (extra_fields) del template

  • Estrazione immagini e grafici dai file HDF5

  • Creazione di file FAIR compatibili integrando metadati eLab

  • Generazione di un report HTML con immagini, grafici e metadati

  • Configurazione personalizzabile tramite file config.json

Requisiti

  • Python 3.8+

  • PyQt6

  • requests

  • h5py

  • matplotlib

  • numpy

Architettura

Lo script è suddiviso in:

🧠 Backend Functions

  • fetch_template_json(template_id): recupera il template da eLabFTW

  • parse_extra_fields(template): estrae metadati, gruppi e risorse

  • extract_and_save_images(hdf5_path): salva immagini preview dai dataset /Images/Image_000 e /Images/Image_001

  • plot_measurements_h5py(...): genera grafici degli spettri O-PTIR e Raman

  • create_experiment(template_id): crea un nuovo esperimento su eLab

  • update_experiment_data(...): aggiorna il body e i metadati dell’esperimento

  • create_fair_ptir_copy(...): genera una copia FAIR del file .ptir con metadati in formato leggibile da HDF5

  • get_ptir_metadata(...): estrae valori singoli, intervalli e liste di attributi dai gruppi di misura

  • create_html_report(...): crea un report leggibile in HTML con anteprima dei dati e metadati

💻 Interfaccia Grafica (PyQt6)

InitialWindow

  • Permette la selezione del file .ptir e del template eLab

MainForm

  • Genera dinamicamente un form basato sui campi extra_fields

  • Permette all’utente di compilare il form e inviarlo

  • Gestisce la validazione, creazione esperimento, patch, salvataggio FAIR e report

Configurazione

Lo script si appoggia a un file config.json nel formato:

json
{
  "BASE_URL": "https://your-elab-instance/api/v2",
  "API_KEY": "your-api-key",
  "TEMPLATES": {
    "Nome Template": ID_NUMERICO
  }
}

Output

  • Report HTML: ptirodat_generated_files/experiment_report.html

  • Immagini: .png estratte da immagini HDF5

  • File FAIR .ptir: con metadati eLab salvati in elab_metadata

Avvio

Lanciare lo script direttamente:

bash
python DMM.py

Note

  • L’accesso ai dati eLab richiede API key attiva.

  • Il parsing dei campi è dinamico: lo script supporta diverse tipologie (text, number, checkbox, items, select, ecc.).

  • I file FAIR sono salvati con suffisso _fair.ptir e possono essere riutilizzati o caricati su altri sistemi.

DMM.py - eLabFTW Automation Tool for PTIR Data

Overview

DMM.py is a Python script with a PyQt6 graphical interface that simplifies the workflow for creating, documenting, and uploading spectroscopy experiments (in .ptir HDF5 format) to the eLabFTW electronic lab notebook. It enables:

  • GUI-driven metadata input

  • Automatic image and spectral preview extraction

  • Metadata parsing and FAIR data generation

  • REST API interaction with eLabFTW

  • HTML report generation for visual inspection

Features

  • 🔍 Select PTIR file and eLabFTW template

  • 🧾 Parse extra_fields and template metadata automatically

  • 🖼️ Extract preview images from HDF5 file datasets

  • 📈 Plot Raman and O-PTIR spectral data

  • 🔁 Create FAIR copy of the .ptir file with integrated metadata

  • 📤 Automatically create and patch experiment records in eLabFTW

  • 📑 Generate a complete HTML report with images and metadata summary

Installation

Requirements:

pip install pyqt6 requests h5py matplotlib numpy

Configuration

A config.json file must be provided in the same folder. Format:

{
"BASE_URL": "https://your-elab-instance/api/v2",
"API_KEY": "your-api-key",
"TEMPLATES": {
"Standard Template": 29
}
}

🔐 Keep your API key private!

Usage

python DMM.py

  1. Select the template and a .ptir file.

  2. Fill in all required metadata fields.

  3. Submit to create the experiment in eLabFTW.

  4. FAIR copy and HTML report are saved automatically.

Output

  • 📁 ptirodat_generated_files/:

    • experiment_report.html: visual HTML summary

    • *_fair.ptir: FAIR version of original file

    • .png preview images from HDF5 dataset

    • Spectral plots (O-PTIR and Raman)
